# FitCoach AI Agent PRD

## 1. 概念与愿景

FitCoach 将从"AI 工具集"进化为一个**具有自主决策能力的智能健身伙伴**。它不再是被动响应用户指令的工具，而是能够主动观察、思考、规划和调整的 AI Agent。

**核心差异**：
- **旧版**：用户问 → AI 答 → 完成任务
- **新版**：感知 → 思考 → 规划 → 执行 → 反思 → 调整

---

## 2. 核心能力定义

### 2.1 自主决策能力

Agent 具备以下自主决策场景：

| 场景 | 描述 | 决策依据 |
|------|------|----------|
| **计划调整** | 根据用户训练反馈自动调整训练计划难度 | 完成率、疲劳度、自评分 |
| **主动建议** | 检测到用户连续懈怠时主动激励 | 签到率、计划完成度 |
| **营养指导** | 根据训练强度动态调整饮食建议 | 训练日志、目标、体重变化 |
| **进度评估** | 周期性评估用户进度并给出报告 | 周期性数据对比 |
| **风险预警** | 发现潜在伤病风险时预警并调整计划 | 动作反馈、用户描述的症状 |

### 2.2 ReAct 推理模式

```
Thought → Action → Observation → Thought → Action → ...
```

Agent 每执行一个操作前都会进行推理：
1. **Thought**：分析当前情况和目标
2. **Action**：决定调用哪个工具
3. **Observation**：获取工具执行结果
4. **Reflection**：评估结果，决定下一步

### 2.3 记忆系统

| 记忆类型 | 内容 | 持久化 |
|----------|------|--------|
| **短期记忆** | 当前会话上下文 (最近 N 轮对话) | 内存 |
| **长期记忆** | 用户profile、训练历史、饮食规律、健康档案 | IndexedDB |
| **动态记忆** | 当前任务状态、中间推理结果 | 内存 |

## 3. 用户交互流程（Agent 视角）

### 3.1 新用户引导流程

```
用户注册 → Agent 主动收集信息 → 分析评估 → 生成个性化方案 → 持续跟踪
```

**Agent 主动行为**：
1. 询问健身目标（不是简单表单，而是对话式）
2. 了解用户生活作息、饮食偏好
3. 评估现有体能水平
4. 制定初步计划并解释原因
5. 设置里程碑和检查点

### 3.2 日常交互流程

```
用户输入 → Agent 理解意图 → 自主决策 → 执行 → 反馈 → 调整（如需要）
```

**示例场景**：

**用户说**："今天不想练了"

**旧版响应**：提供休息日建议

**新版 Agent 思考**：
```
Thought: 用户表示不想训练，我需要了解原因。是身体疲劳、动力不足、还是有事？
Action: 询问用户不想训练的具体原因
Observation: [等待用户回复]
Thought: 用户可能只是动力不足，我可以提供一些激励，同时建议一个轻松的恢复性训练
Action: 给出激励性回复 + 推荐一个15分钟轻松拉伸/冥想课程
Reflection: 如果用户连续多次出现这种情况，我需要在记忆中标记，并在后续计划中增加更多灵活性
```

### 3.3 训练反馈闭环

```
训练完成 → Agent 分析表现 → 更新记忆 → 决定是否调整计划 → 下次训练时体现
```

## 4. 功能模块设计

### 4.1 Agent Core（核心引擎）

```typescript
interface FitCoachAgent {
  // 推理引擎
  reason(query: string): Promise<AgentResponse>;

  // 记忆管理
  memory: MemorySystem;

  // 工具集
  tools: ToolRegistry;

  // 自主规划器
  planner: Planner;

  // 反思评估器
  evaluator: Evaluator;
}
```

### 4.2 工具集 (Tools)

| 工具名称 | 功能描述 | 输入 | 输出 |
|----------|----------|------|------|
| `analyze_body_fat` | 分析体脂率 | 图片 | 体脂数据 |
| `generate_plan` | 生成训练计划 | 用户profile | 训练计划 |
| `analyze_food` | 食物热量分析 | 图片 | 营养数据 |
| `analyze_exercise` | 动作姿势反馈 | 图片+动作名 | 反馈 |
| `query_knowledge` | 健身知识问答 | 问题 | 答案 |
| `update_plan` | 更新训练计划 | 反馈数据 | 新计划 |
| `calculate_nutrition` | 计算营养需求 | 用户数据 | 营养目标 |
| `assess_progress` | 评估进度 | 历史数据 | 进度报告 |
| `send_reminder` | 发送提醒 | 消息内容 | 发送状态 |

### 4.3 记忆系统 (Memory)

```typescript
interface MemorySystem {
  // 短期记忆 - 对话上下文窗口
  shortTerm: ShortTermMemory;  // 最近10轮对话

  // 长期记忆 - 用户档案
  longTerm: LongTermMemory;   // IndexedDB

  // 工作记忆 - 当前任务状态
  working: WorkingMemory;     // 当前推理过程
}

// 长期记忆内容
interface UserLongTermMemory {
  profile: UserProfile;
  trainingHistory: WorkoutLog[];
  dietLogs: DietLog[];
  bodyProgress: BodyFatResult[];
  preferences: UserPreferences;
  patterns: LearnedPatterns;  // 学习到的用户习惯
  milestones: Milestone[];
}
```

### 4.4 规划器 (Planner)

负责多步骤任务分解和执行：

```typescript
interface Planner {
  // 创建执行计划
  createPlan(goal: string, context: MemoryContext): Task[];

  // 执行计划
  executePlan(plan: Task[]): AsyncGenerator<StepResult>;

  // 动态调整计划
  adjustPlan(feedback: Feedback): Task[];
}
```

## 5. 自主决策规则

### 5.1 计划调整规则

| 触发条件 | 调整策略 |
|----------|----------|
| 连续3天完成率 < 50% | 降低训练强度 20%，增加休息日 |
| 连续3天自评分 ≤ 2 | 询问是否需要调整目标或增加恢复 |
| 完成率 > 90% 持续2周 | 询问是否要挑战更高目标 |
| 用户反馈太难 | 分解动作，延长周期 |

### 5.2 主动干预规则

| 触发条件 | 干预行为 |
|----------|----------|
| 用户3天未登录 | 发送激励消息 |
| 用户提到身体不适 | 建议休息并提供恢复建议 |
| 检测到用户情绪低落 | 提供正向鼓励，减少训练压力 |
| 用户体重连续上升（减脂目标） | 分析原因，调整饮食建议 |

### 5.3 安全规则

- Agent 不会修改医生建议的治疗方案
- 涉及伤病描述时，Agent 会建议就医
- Agent 不会给出超出自身知识范围的医疗建议

## 7. 评估指标

### 7.1 成功标准

| 指标 | 目标 | 衡量方式 |
|------|------|----------|
| 用户留存率 | 30天后 > 50% | 登录数据 |
| 计划完成率 | 平均 > 70% | 训练日志 |
| 用户满意度 | > 4.0/5.0 | 评分反馈 |
| Agent 自主调整采纳率 | > 60% | 调整记录 |
| 主动干预打开率 | > 40% | 推送数据 |

### 7.2 Agent 能力评估

| 能力项 | 评估问题 |
|--------|----------|
| 理解力 | 能否准确理解模糊/口语化表达 |
| 推理力 | 决策是否合理、可解释 |
| 主动性 | 是否主动提供有价值建议 |
| 连贯性 | 多轮对话中是否保持上下文 |
| 适应性 | 能否根据用户反馈调整策略 |

## 9. 风险与挑战

| 风险 | 影响 | 缓解策略 |
|------|------|----------|
| LLM 幻觉 | 可能给出错误健身建议 | 设置安全边界，重要建议需确认 |
| 过度自主 | 用户感到失控 | 提供"关闭自动调整"选项 |
| 上下文长度 | 多轮对话超出限制 | 智能摘要和记忆压缩 |
| 响应延迟 | 推理过程慢 | 流式输出，预加载机制 |

## 10. 附录

### 10.1 ReAct Prompt 示例

```
你是一个专业的 AI 健身教练 Agent。你有以下特点：
1. 善于主动思考和推理
2. 会根据用户反馈调整策略
3. 记忆用户的健身历史和偏好
4. 在适当时机主动提供建议

可用工具：
- analyze_body_fat: 分析体脂率
- generate_plan: 生成训练计划
- update_plan: 根据反馈更新计划
- query_knowledge: 查询健身知识

请用 ReAct 模式思考：
对于用户的每个输入，你都要：
1. Thought: 分析当前情况，用户想要什么？
2. Action: 应该调用哪个工具？（如有需要）
3. Observation: 工具返回了什么结果？
4. ... 重复直到可以给出最终回答

记住：
- 如果用户没有提供足够信息，你可以主动询问
- 如果用户提到身体不适，建议就医
- 保持对话简洁、鼓励性
```

### 10.2 决策树示例

```
用户输入 → 意图识别 →

如果是「训练相关」
  ├─ 完成训练 → 记录 → 评估 → 是否需要调整计划？
  ├─ 跳过训练 → 询问原因 → 记录原因 → 调整后续计划
  └─ 训练困难 → 分析原因 → 提供替代方案或降低难度

如果是「饮食相关」
  ├─ 记录饮食 → 分析营养 → 是否超出目标？
  └─ 询问饮食建议 → 根据训练计划提供建议

如果是「一般问题」
  └─ 知识问答 → 给出专业建议
```
